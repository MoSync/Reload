mosync.resource.loadImage

Loads images into image handles for use in MoSync UI systems.

Parameters:

  • imagePath relative path to the image file.
  • imageID a custom ID used for refering to the image in JavaScript

  • callBackFunction a function that will be called when the image is ready.

    Example

    mosync.resource.loadImage("./img/logo.png", "Logo", function(imageID, imageHandle){
            var myImageButton = document.getNativeElementById("myImageButton");
            myImageButton.setProperty("image", imageHandle);
        });

mosync.resource.loadRemoteImage

Loads images into image handles from a remote URL for use in MoSync UI systems.

Parameters:

  • imageURL URL to the image file.
  • imageID a custom ID used for refering to the image in JavaScript

  • callBackFunction a function that will be called when the image is ready.

    Example

    mosync.resource.loadRemoteImage("http://www.example.com/img/logo.png", "Logo", function(imageID, imageHandle){
            var myImageButton = document.getNativeElementById("myImageButton");
            myImageButton.setProperty("image", imageHandle);
        });

mosync.resource.sendRemoteLogMessage

Send a log message to a remote server. This is useful for displaying debug info when developing/testing on a device.

When using Reload, the Reload Client is set up to send log messages to the Reload Server. Nothing needs to be written or configured in C++ in this case.

If you wish to implement your own logging handler, you do this in C++ by creating a subclass of class Wormhole::LogMessageListener, implementing method onLogMessage, and then setting the listener using Wormhole::ResourceMessageHandler::setLogMessageListener(). See the implementation of the Reload Client for an example of this.

Parameters:

  • message The message to be sent, for example "Hello World".

  • url Optional string parameter that specifies url of the remote server that should handle the log request, for example: "http://localhost:8282/remoteLogMessage/" If this parameter is not supplied or set to null, "undefined" will be passed to the C++ log message handler.

mosync.rlog

Short alias for mosync.resource.sendRemoteLogMessage ("rlog" is short for "remote log"). If you use Reload, you can call mosync.rlog like this: mosync.rlog("Hello World"); The log message will show up in the Reload user interface.
